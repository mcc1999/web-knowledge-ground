# 字符编码——ASCII、GBK、Unicode、UTF-8

## ASCII

American Standard Code for Information Interchange，美国信息交换标准代码, 最早用 8 位二进制来表示英文字符、数字和符号，最前面一位为 0.

## GBK

由于 ASCII 编码无法满足在计算机系统中表示中文，GBK 在 GB2312 基础上扩展了字符集以支持简体中文，兼容 ASCII。

## Unicodeeeeeeeeeeeeeeeeeeeeeeee

除了中文、英文，世界上还有其他语言如法语、阿拉伯语等，需要一个更大字符集来支持所有语言字符，Unicode 应运而生。它给世界上所有字符都分配了一个唯一编号，从 `0x000000 到 0x10FFFF`

## UTF

Unicode Transformation Format, 表示一组将 Unicode 字符编码为二进制的标准格式。

### UTF-32

每个 Unicode 字符用 4 个字节，32bit 保存。缺点：对于 0x1 这样的小字符，占用空间过大。

```javascript
/**
 * "马"的Unicode码为 0x9A6C，换为整数是39532，二进制为 1001 1010 0110 1100
 * 采用4字节保存
 * 00000000 00000000 10011010 01101100，大端：00 00 9A 6C，小端：6C 9A 00 00
 */
```

### UTF-16

- 采用可变字节。
- `0x0000 ～ 0xFFFF`，使用 2 个字节，16bit 保存；
- `0xFFFF ～ 0x10FFFF`，使用 4 个字节，32bit 保存；

```javascript
/**
 * "马"的Unicode码为 0x9A6C，换为整数是39532，二进制为 1001 1010 0110 1100
 * 采用2个字节保存
 * 10011010 01101100，大端：9A 6C，小端：6C 9A
 */
```

### UTF-8

- 采用 1-4 个可变字节。
- 0~127,采用 1 个字节，8bit 保存，最前面一位为 0；
- 对于 n 个字节保存的字符，n 个字节中的第一个字节前 n 位为 1，第 n+1 位为 0，后面剩下的每个字节前两位均为 10，剩下的位由这个字符的二进制位从左到右被填充。

```javascript
/**
 * "马"的Unicode码为 0x9A6C，换为整数是39532，二进制为 1001 1010 0110 1100
 * 采用3字节保存
 * 1110XXXX 10XXXXXX 10XXXXXX
 * 11101001 10101001 10101100，十六进制：E9 A9 AC
 */
```
